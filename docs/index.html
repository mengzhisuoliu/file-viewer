纯前端 Serverless

不依赖后端转码服务，浏览器直接完成解析和渲染，适合静态部署、私有化部署和离线场景。

59 种扩展名覆盖

覆盖 Office、PDF、OFD、CAD、Markdown、代码/文本、图片和视频，完整承接常见附件中心需求。

按需异步加载

OFD、CAD、PDF、Office、Markdown 和代码高亮链路按格式拆包，重型依赖不拖慢其他文件首屏。

优势一览 ​

少一条服务端链路

大多数格式直接在浏览器完成解析，不需要把附件交给后端转 PDF，也减少了临时文件、任务队列和清理脚本的维护成本。

格式覆盖更贴近业务

除了 Office、PDF 和图片，也补齐 OFD、DXF、Markdown、代码、日志、配置文件和 MP4，适合 OA、知识库、工单和附件中心。

重型能力不抢首屏

PDF、OFD、CAD、Office 和代码高亮都按需异步加载。用户打开图片或文本时，不会因为图纸或版式文档解析器而额外等待。

阅读质感有兜底

Word 保留白色纸张和灰色页面底，PDF 具备缩放、页码、导航窗格和宽度自适应，长文档打开后默认就应该可读。

接入边界清晰

Vue3 与 Vue2.7 都有对应 npm 包，iframe 方案适合多系统复用。URL、File、Blob 转 File 等输入路径在文档里都有示例。

成品可直接下载

公开仓库提供混淆压缩库产物、Demo 静态站点、文档静态站点、样例文件和 tarball，源码需求可通过官方渠道自助开通。

最快接入路径 ​

我想先看效果

打开 在线 Demo，用内置样例或本地文件验证格式、缩放、导航和样式表现。

我在 Vue 3 项目里用

安装 @flyfish-group/file-viewer3@1.0.7，再查看 Vue3 集成 接入 URL 或 File 输入。

我还在 Vue2 项目里用

安装 @flyfish-group/file-viewer@1.0.7，按 Vue2 集成 使用 Vue.use(FileViewer) 注册。

我要给多个系统共用

阅读 Iframe 嵌入，把预览器作为独立静态站部署，通过 URL 或 postMessage 推送二进制文件。

我要下载成品或二开

查看 发布与成品分发。公开仓库提供可下载产物，源码和商业二开通过官方渠道开通。
